Описание структуры проекта



Проект Fifi-Study — мобильное Android-приложение для обучения, взаимодействия пользователей, выполнения заданий и общения.
Разработка выполнена на Java + Android SDK с клиент-серверным взаимодействием через REST API.



activities/ (экраны приложения)



Папка содержит Activity-классы отвечает за пользовательский интерфейс и навигацию между экранами.

Основные экраны:

MainActivity — главный экран приложения
MainActivity_profile — профиль пользователя
MainActivityPalka — навигационный экран
MainActivity_friend — список друзей
FriendRequestsActivity — входящие заявки в друзья

Экраны обучения:

PracticeStudyActivity
PRACTICA, PRACTICA_SOVMEST
MainTaskActivity
activity_theory, activity_practice

Авторизация и доступ:

MainSecretKode, MainSecretKode_VXOD
MainZabilParol — восстановление пароля
activity_reg, activity_in — регистрация и вход

Гостевой режим реализован через GuestResponse

Дополнительные экраны:

RatingActivity — рейтинг пользователей
ObhiChatActivity — общий чат
Main_Nastroiki — настройки
MainTHEME — тема оформления



res/layout/ (XML-разметка)



XML-файлы интерфейсов для экранов приложения.

activity_main.xml — главный экран
activity_profile.xml — профиль пользователя
activity_profile_palka.xml — боковое меню
activity_practice_study.xml — экран практики
fragment_friend_requests.xml — фрагмент заявок

Интерфейс реализован в минималистичном карточном стиле с поддержкой навигации.



adapters/ (адаптеры списков)



Используются для отображения списков данных (RecyclerView, ListView):

TasksAdapter — задания
MessagesAdapter — сообщения
FriendRequestsAdapter — заявки в друзья
RoleAdapter — роли пользователей

Адаптеры отделяют логику отображения от бизнес-логики.



models/ (модели данных)



Классы, представляющие данные, получаемые от сервера или используемые в приложении:

UserProfileResponse
UserPhotoResponse
RatingUser
Task
Theory
MessageModel
InviteModel

Используются для работы с REST API и передачи данных между слоями приложения.



network/ (работа с сервером)



Компоненты для сетевого взаимодействия:

RetrofitClient — настройка клиента Retrofit
RequestLogin, RequestRegister — запросы
ResponseUser, ResponseGoogleLogin — ответы сервера
Приложение использует REST API для:

авторизации
получения профиля
загрузки фото
работы с заданиями
общения между пользователями



cache/ и утилиты



UserCache — хранение данных пользователя в памяти
ImageCache — кэш изображений
UserPhotoLoader — загрузка фото
NotificationUtils — уведомления

Используются для уменьшения сетевых запросов.



services / receivers



InviteCheckerService — проверка приглашений
InviteReceiver, InviteActionReceiver — обработка событий
Локальные уведомления реализованы через BroadcastReceiver
